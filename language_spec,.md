# Timeseries Intermediate Language (TSIL)

**TSIL** is a domain-specific language (DSL) designed for financial timeseries analytics. In derivatives trading the timeseries may be implied vol, realised vol, spot, percentage change in spot, rate, dividend yields, borrows etc. The language can express complex operations on timeseries (for e.g timeseries1 + 0.5 * sqrt(timeseries2)).

**Why**
TSIL creates a standardised, portable specification for the required data much like any language spec such as C++, python etc. This repo also contains an interpreter for this language.

Pros:

1. Intuitive - TSIL syntax is intuitive and can be written by users (or generated by LLM agents from natural-language prompts).
2. No hallucination - LLM agent with TSIL language spec is less likely to hallucinate.
3. Save/Store - TSIL script (or AST) can be saved and refreshed as required without going through the agent each time.
4. Parallelisation - A compiled TSIL code (AST) can be executed locally by an in-memory runtime engine or they can be parallelised by an appropriate distributed runtime.
5. Debugging - Its easier to debug an actual script as opposed to black-box LLM output.

# Key Features

- **Intuitive Syntax**: Write complex financial queries in a readable, concise manner
- **Composable Objects**: Build complex expressions from simple, reusable building blocks
- **Market-Aware**: Native support for financial concepts like tickers, expiries, strikes, implied/realised vol etc
- **Type-Safe**: Clear object types prevent common data manipulation errors

# Variables (Storage)

Variables are dynamically typed and can assume a fundamental type or an object. Objects have fields and methods.

**Examples**

```python
spx = t("SPX") # spx is a ticker object
spx = IV(spx, e("3M"), k("25D")) # spx is now a timeseries
```

# Fundamental Types

## Boolean

Booleans are True|False flags.

## String

String literals represent text values and enclosed in "".

**Examples:**

```text
"SPX"
"3M"
"100%"
"DEC2026"
```

## Number

Both integers and floating-point numbers are supported for numeric computations.

**Examples:**

```text
100
25.5
0.75
```

## Date

Dates must follow the ISO 8601 format: `YYYY-MM-DD`. For conciseness there is no other supported format.

**Examples:**

```text
2026-12-17
2025-06-01
2024-01-15
```

## Datetime

Datetime must follow the ISO 8601 format: `YYYY-MM-DDTHH:mm:ss`. For conciseness there is no other supported format.

**Examples:**

```text
2026-12-17THH:mm:ss
2025-06-01THH:mm:ss
2024-01-15THH:mm:ss
```

# Objects

## List

Lists are ordered collections of values, enclosed in square brackets. The elements of List can be any supported type. The List can be of mixed types.

**Syntax:**

```
[x1, x2, ..., xn]
```

**Examples:**

```
["SPX", "SX5E"]
[0.3, 0.7]
[1, 2, 3, 4, 5]
```

## Timeseries

A **Timeseries** is a time-ordered/indexed list of values with some meta-data. The language does not enforce the underlying datastructure but the runtime will likely use pandas Series object. Operations on series objects will be supported.

**Value Type**

- vol: vol level (e.g 0.13 for 13vol)
- vol100: vol level scaled by 100 (e.g 13)
- pct: percentage (e.g 0.43 for 43%)
- pct100: percentage scaled by 100 (e.g 43)
- price

**Structure**

```json
{
  id: "...", // a system generated id
  name: "SPX 3M Atm Vol", // A user-friendly name e.g for use in a plotting tool
  value_type: "vol",
  values: [
    {timestamp: 2026-12-17T01:00:00, value: 100.01},
    {timestamp: 2026-12-17T01:00:01, value: 100.05},
    ...
  ]
}
```

## Ticker

A **Ticker** object represents a symbol (stock, index, fx, future contract etc) with some information about the symbol such as currency, exchange etc. They are constructed by Bloomberg IDs (bbg, bbpk) or Reuters codes (ric). Tickers can also represent a weighted basket of symbols.

**Factory method (t)**

Factory function `t` creates a ticker object.

**Syntax:**

```
t(ticker_list, [weights=WGT_EQ])
```

**Parameters:**

- `ticker_list`: Single ticker string or list of ticker strings
- `weights` (optional): Weighting scheme enum or list of numeric weights. Defaults to `WGT_EQ`

**Weighting Schemes:**

- `WGT_EQ` – Equal-weighted across all instruments
- `WGT_VOL` – Volatility-weighted (inverse volatility weighting)
- `WGT_MOM` – Momentum-weighted
- `WGT_MCAP` – Market-cap weighted

**Examples:**

```python
# Single ticker
t1 = t("SPX") # using bbg (short Bloomberg key)
t2 = t("SPX Index") # using bbpk (full Bloomberg key)
t3 = t(".SPX")  # using ric (Reuters code)

# Equal-weighted basket (default)
basket1 = t(["SPX", "SX5E"]) # using equal weighting by default
basket1 = t(["SPX", "SX5E"], WGT_EQ)  # Equivalent to above

# Custom-weighted basket
basket2 = t(["SPX", "SX5E"], [0.3, 0.7])  # SPX 30%, SX5E 70%

# Volatility-weighted basket
basket3 = t(["SPX", "SX5E", "MSFT.O"], WGT_VOL)

# Momentum-weighted basket
basket4 = t(["SPX", "SX5E"], WGT_MOM)
```

## Expiry

An **Expiry** object specifies the maturity date or tenor for derivatives (options, futures). TSIL supports two expiry types: fixed-date expiries and constant-maturity expiries (tenors). It can also express forward periods (e.g 6M1Y => 1 year expiry in 6-month time).

**Factory method (t)**

Factory function `e` creates an expiry object.

**Syntax:**

```
e(expiry, [duration])
```

**Parameters:**

- expiry : A valid tenor or a fixed expiry
- duration (optional): For forward periods, a duration tenor or a fixed expiry

### Fixed Expiry

Fixed expiries refer to a specific calendar date or contract month. When only month and year is provided system refers to standard monthly expiry date.

**Formats:**

- **ISO Date**: `e("2026-12-17")`
- **Month-Year**: `e("DEC2026")` or `e("DEC26")`
- **Listed Code**: `e("Z26")` (CME-style contract codes)

**Listed Monthly Expiry Codes:**

```
F = January      M = June         X = November
G = February     N = July         Z = December
H = March        Q = August
J = April        U = September
K = May          V = October
```

**Fixed Expiry Examples:**

```python
e_iso = e("2026-12-17")        # ISO date format
e_month = e("DEC2026")         # Month-Year format
e_code = e("Z26")              # Listed expiry code (December 2026)
e_short = e("JUN25")           # Short month format (June 2025)
```

### Tenor

Tenors are constant-maturity (relative) expiries. At each point in time, the metric refers to an instrument expiring at a constant number of business days (e.g., always 3 months later).

**Tenor Format:** `{UNITS}{PERIOD}`

**Period Codes:**

- `D` – Business days
- `B` – Business days (alternative)
- `W` – Weeks
- `M` – Months
- `Y` – Years

**Tenor Examples:**

```python
e_3m = e("3M")      # 3-month tenor
e_1y = e("1Y")      # 1-year tenor
e_1w = e("1W")      # 1-week tenor
e_2d = e("2D")      # 2 business days
e_18m = e("18M")    # 18-month tenor
```

### Forward Expiry

Forward expiries starting at a future date and expiring after a specified duration. Use two parameters: start date and duration. When used with Implied Vol metric its calculates forward volatility.

**Forward Expiry Examples:**

```python
# 1-year forward starting 6 months out (1Y6M forward thus start end end expiries roll every day in the timeseries)
e_fwd1 = e("1Y", "6M")

# Z contract (Dec 2026) forward 6 months (start and end expiry remain fixed in the timeseries
e_fwd2 = e("Z26", "6M")

# Z6 forward expiring at Z7
e_fwd3 = e("Z6", "Z7")

# 2-year forward starting 1 year out
e_fwd4 = e("1Y", "2Y")
```

### Strike

**Strike** object represents the exercise price for an option. Strikes can be specified as absolute prices or as relative measures (delta, moneyness, normalized levels).

**Factory method (t)**

Factory function `k` creates a Strike object.

**Syntax:**

```
k(level, [strike_type])
```

**Parameters:**

- `level`: strike value (can be number for absolute strike or a string for other types)
- `strike_type` (optional): strike type ("%","%F","%S","D","DP","DC","N"). String strike type can often be implied by the level paramter as the level param can directly take the strike type:

**Strike Types**

| Type              | Indicator   | Meaning                                     | Example     |
| ----------------- | ----------- | ------------------------------------------- | ----------- |
| Absolute          | (none)      | Strike price in absolute terms              | `k(7500)`   |
| Forward Moneyness | `%` or `%F` | Strike as % of forward price (100% = ATM)   | `k("100%")` |
| Spot Moneyness    | `%S`        | Strike as % of spot price                   | `k("95%S")` |
| Delta (Auto)      | `D`         | Delta specification (+ for call, - for put) | `k("25D")`  |
| Put Delta         | `DP`        | Put delta (converted to absolute)           | `k("25DP")` |
| Call Delta        | `DC`        | Call delta (converted to absolute)          | `k("25DC")` |
| Normalized        | `N`         | Normalized strike level                     | `k("1.5N")` |

**Strike Examples**

```python
# Absolute strikes
k1 = k(7500)              # 7500 absolute strike
k2 = k(100.5)             # 100.5 absolute strike

# Moneyness-based
k_atm = k("100%")         # 100% forward moneyness (at-the-money)
k_otm = k("95%")          # 95% forward moneyness (out-of-the-money)
k_spot = k("100%S")       # 100% spot moneyness

# Delta-based
k_delta_call = k("25D")   # 25 delta (interpreted as call if positive)
k_delta_put = k("-25D")   # -25 delta (put)
k_dp = k("25DP")          # 25 delta put
k_dc = k("25DC")          # 25 delta call

# Normalized
k_norm = k("1.5N")        # 1.5 normalized strike

# Using explicit strike type
k_dp = K(25,"dp")
k_norm = k(1.75, "N")
```

# Metrics

**Metrics** are functions that return specific types of timeseries data—spot prices, implied volatilities, realized volatilities, correlations, Greeks, and custom signals. Each metric is generated by applying a metric function to instrument and optional derivative parameters.

## Implied Volatility (IV)

**Implied Volatility** represents the market's expectation of future volatility as implied by option prices. IV is a key metric for trading, risk management, and valuation.

**Syntax:**

```
IV(ticker, expiry, strike, **kwargs)
```

**Parameters:**

- `ticker`: Ticker object `t(...)`
- `expiry`: Expiry object `e(...)`
- `strike`: Strike object `k(...)`
- 'kwargs': named parameters (e.g fixed_k_chg=True, roll_expiry=True)

**Returns:**

- A timeseries of implied volatility values indexed by datetime

### Examples

**Example 1: Simple Single-Ticker IV**

```python
# SX5E (Stoxx 50 Index) 3-month ATM implied volatility
iv1 = IV(t("SX5E"), e("3M"), k("100%"))
```

**Example 2: Basket IV with Custom Weights**

```python
# Weighted average of SX5E and SPX 3-month ATM vol
# SX5E = 60% weight, SPX = 40% weight
iv2 = IV(t(["SX5E", "SPX"], [0.6, 0.4]), e("3M"), k("100%"))
```

**Example 3: Delta Call IV with Fixed Expiry**

```python
# SX5E 25 delta call on December 2026 contract
iv3 = IV(t("SX5E"), e("Z26"), k("25DC"))
```

**Example 4: Volatility-Weighted Basket IV**

```python
# Volatility-weighted basket (inverse vol weighting)
# 1-year tenor, ATM
iv4 = IV(t(["SPX", "SX5E", "MSFT.O"], WGT_VOL), e("1Y"), k("100%"))
```

**Example 5: Forward IV**

```python
# 1-year implied volatility for 6-month forward
# (Vol for contracts 1Y-6M forward)
iv5 = IV(t("SX5E"), e("1Y", "6M"), k("100%"))
```

**Example 6: Out-of-the-Money IV**

```python
# 10% out-of-the-money put IV for 3 months
iv6 = IV(t("SPX"), e("3M"), k("90%"))
```

## Cost of carry

### Gamma Carry

Theta cost per day adjusted by vol roll (for atm straddle of given expiry).

***Syntax:***

```
GC(ticker, expiry)
```

**Parameters:**

- `ticker`: Ticker object `t(...)`
- `expiry`: Expiry object `e(...)`

**Returns:**

- A timeseries of implied cost of carry indexed by datetime

### Vanna Carry

Theta cost per day adjusted by vol roll (for 25D straddle flat gamma of given expiry).

***Syntax:***

```
VC(ticker, expiry)
```

**Parameters:**

- `ticker`: Ticker object `t(...)`
- `expiry`: Expiry object `e(...)`

**Returns:**

- A timeseries of implied cost of carry indexed by datetime

### Volga Carry

Theta cost per day adjusted by vol roll (for 10D strangles flat gamma of given expiry).

***Syntax:***

```
VGC(ticker, expiry)
```

**Parameters:**

- `ticker`: Ticker object `t(...)`
- `expiry`: Expiry object `e(...)`

**Returns:**

- A timeseries of implied cost of carry indexed by datetime

## Backtest

TSIL provides some templatised backtesting mechanism. For more bespoke backtests use a sophisticated backtester such as eqdlib.

### Instruments

There are following set of instruments provided:

- CALL(ticker, expiry, strike, size) : Vanilla call option.
- PUT(ticker, expiry, strike, size) : Vanilla put option.
- SD(ticker, expiry, size) : atm-forward straddle
- RR(ticker, expiry, strike, size) : given delta risk reversal (long put short call)
- PS(ticker, expiry, delta, ratio, size) : put spread
- CS(ticker, expiry, delta, ratio, size) : call spread
- CCS(ticker, expiry, strike, size) : Call calendar spread
- PCS(ticker, expiry, strike, size) : Call calendar spread

**Example**

```python
t1 = t("SPX")
e1 = e("3M")
e2 = e("1Y")
ptf = [SD(t1, e1), SD(t1, e2)]
sz = notional(100000, "vega")
sig = IV(t1, e("1Y"), k("100%")) - IV(t1, e("1M"), k("100%"))
sig = (sig > ma(sig, 20))
res = bt(ptf, sz, "weekly") * sig
```

## Operations (on Timeseries)

The following operations are supported on timeseries:

### Arithmetic Operations

| Operator        | Name           | Description                 |
| --------------- | -------------- | --------------------------- |
| `+`             | Addition       | Element-wise addition       |
| `-`             | Subtraction    | Element-wise subtraction    |
| `*`             | Multiplication | Element-wise multiplication |
| `/`             | Division       | Element-wise division       |
| `**` or `pow()` | Power          | Element-wise power          |

**Examples:**

```python
# Add two timeseries
t1 + t2

# Multiply by a constant
t1 * 2

# Calculate annualized vol (assuming daily data)
annualized_vol = t1 * sqrt(252)

# Percentage change
percentage_change = (t1 - t2) / t2 * 100

# Vol spread
iv_spd = IV(t("SX5E"), e("2Y"), k("100%")) - IV(t("SX5E"), e("1Y"), k("100%"))

# Vol ratios
iv_spd = IV(t("SX5E"), e("2Y"), k("100%")) - 1.5 * IV(t("SX5E"), e("1Y"), k("100%"))
```

---

### Mathematical Functions

| Function                    | Description                            |
| --------------------------- | -------------------------------------- |
| `sqrt(ts)`                  | Square root of ts                      |
| `diff(ts, periods=1)`       | Difference of ts by periods            |
| `pct_change(ts, periods=1)` | Percentage change of ts by periods     |
| `corr(ts, ts2, window)`     | Rolling correlation between ts and ts2 |
| `cov(ts, ts2, window)`      | Rolling covariance between ts and ts2  |
| `std(ts, window)`           | Rolling standard deviation of ts       |
| `mean(ts, window)`          | Rolling mean of ts                     |
| `sharpe(ts, window)`        | Rolling Sharpe ratio of ts             |
| `sum(ts, window)`           | Rolling sum of ts                      |
| `min(ts, window)`           | Rolling minimum of ts                  |
| `max(ts, window)`           | Rolling maximum of ts                  |
| `mode(ts)`                  | Mode of ts                             |
| `percentile(ts, q)`         | Percentile of ts                       |
| `drawdown(ts, window)`      | Drawdown of ts                         |

**Examples:**

```python
# 3-month rolling Sharpe ratio
rolling_sharpe = sharpe(ts1, window=360)

# 30-day rolling correlation between two timeseries
rolling_corr = corr(ts1, ts2, window=30)

# Historical drawdown
hd = drawdown(ts1, window=360)
```

## Putting It Together: Complete Workflow Examples

### Example 1: Analyzing Single Index Volatility Term Structure

```python
# Define three points on the volatility term structure for the Stoxx 50 Index
sx5e = t("SX5E")

vol_1m = IV(sx5e, e("1M"), k("100%"))   # 1-month ATM vol
vol_3m = IV(sx5e, e("3M"), k("100%"))   # 3-month ATM vol
vol_1y = IV(sx5e, e("1Y"), k("100%"))   # 1-year ATM vol

# Analyze term structure slope
term_structure_slope = vol_1y - vol_1m
```

### Example 2: Multi-Asset Basket Analysis

```python
# Create a volatility-weighted basket of major indices
basket = t(["SPX", "SX5E", ".STOXX50E", "MSFT.O"], WGT_VOL)

# Get December 2026 25 delta call IV
basket_iv = IV(basket, e("Z26"), k("25DC"))

# Compare to simple equal-weighted basket
eq_basket = t(["SPX", "SX5E", ".STOXX50E", "MSFT.O"], WGT_EQ)
eq_iv = IV(eq_basket, e("Z26"), k("25DC"))

# Difference due to weighting scheme
weighting_impact = basket_iv - eq_iv
```

### Example 3: Skew Analysis

```python
# Analyze the volatility skew (IV smile) on S&P 500 options
spx = t("SPX")

atm_vol = IV(spx, e("1M"), k("100%"))        # At-the-money
otm_put_vol = IV(spx, e("1M"), k("90%"))     # 10% OTM put
otm_call_vol = IV(spx, e("1M"), k("110%"))   # 10% OTM call

# Calculate skew
put_call_skew = otm_put_vol - otm_call_vol
```

### Example 4: Forward Vol Analysis

```python
# Analyze 1-year forward 6-month implied volatility
sx5e = t("SX5E")

# Spot 6-month vol
spot_vol_6m = IV(sx5e, e("6M"), k("100%"))

# 1-year forward 6-month vol
fwd_vol = IV(sx5e, e("1Y", "6M"), k("100%"))

# Forward premium/discount
fwd_premium = fwd_vol - spot_vol_6m
```

## Best Practices

1. **Use Tickers Consistently**: Stick to Bloomberg IDs or Reuters codes; don't mix conventions
2. **Chain Operations**: Build complex analyses from simple, testable components
3. **Weighting Schemes**: Understand the impact of weighting—use equal-weighting as baseline
4. **Tenor vs. Fixed Expiry**: Use tenors for rolling analysis; use fixed expiries for specific contract snapshots
5. **Strike Conventions**: Use delta conventions for options; absolute strikes for spot/forward prices
